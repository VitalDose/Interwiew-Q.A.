Тестирование -  процесс исследования ПО с целью получения информации о качестве продукта;
SOAP - протокол передачи данных в формате XML
Особенности: наличие WSDL, работает с различнимы протоколами, не может быть кеширован, это протокол.
XSD - схема которая описывает структуру XML документа и список даных которые в ней хранятся
WSDL - описывает структуру нашего сервиса
REST - архитектурный стиль с помощью которого описывается структура передачи информации (в стиле не используют WSDL у него меньше правил)
RESTful - веб-сервис которых отвечает требованиям REST
Smoke - самое важное.  Выборка из общего числа запланированных тестовых сценариев, покрывающая основную функциональность компонента или системы. Проводится с целью удостовериться, что базовые функции программы в целом работают корректно, без углубления в детали
Critical path - повседневное. Тесты критического пути запускаются для проверки функциональности, используемой типичными пользователями в их повседневной деятельности. 
Extended - все. Выполняется для изучения всей функциональности, указанной в требованиях. Проверяется даже функциональность с низким приоритетом.
Функциональное тестирование – один из видов тестирования, направленного на проверку соответствий функциональных требований ПО к его реальным характеристикам.

Smoke-тестирование 
Модульное тестирование (Unit testing) – тестирование каждой атомарной функциональности приложения отдельно, в искусственно созданной среде.
Интеграционное тестирование – вид тестирования, при котором на соответствие требований проверяется интеграция модулей, их взаимодействие между собой, а также интеграция подсистем в одну общую систему.
Системное тестирование – это тестирование программного обеспечения выполняемое на полной, интегрированной системе, с целью проверки соответствия системы исходным требованиям, как функциональным, так и не функциональным.
Регрессионное тестирование – это набор тестов, направленных на обнаружение дефектов в уже протестированных участках приложения.
Приемочное (acceptance) – вид тестирования, проводимый на этапе сдачи готового продукта (или готовой части продукта) заказчику.
